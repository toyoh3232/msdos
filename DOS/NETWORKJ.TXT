NETWORKJ.TXT

このテキストファイルでは, お使いのネットワークと Microsoft(R) MS-DOS(R) 
operating system Version 6.2/V との互換性を保つ方法について説明します. 
MS-DOS Version 5.0/V をお使いの方は, MS-DOS 6.2/V と互換性を持たせるためにネッ
トワークに修正を加える必要はありません.

セットアップ中, このテキストファイルを参照するよう指示された場合は, 次のセク
ション 1 を参照してください. お使いのネットワークソフトウェアについての詳細
を知りたい場合は, 下記から該当するセクションを探してください.

このテキストファイルは次のセクションから構成されています.

1　セットアップ中 NETWORKJ.TXT を参照するように指示された場合
=============================================================
セットアップ中「コンピュータがネットワークと接続するよう指定されました」とい
うメッセージが表示された場合は, お使いのコンピュータが MS-DOS 6.2/V と互換性
のない旧バージョンのネットワークソフトウェアを実行している可能性があります. 
詳しくは, 本ファイル内のネットワークソフトウェアに関する部分を参照してくださ
い. お使いのネットワークソフトウェアに MS-DOS 6.2/V との互換性を持たせる方法
が示されていない場合は, そのソフトウェアのメーカーに問い合わせて, ソフトをバ
ージョンアップしてください. その後で, バージョンアップしたネットワークファイ
ルを組み込み, 再び MS-DOS 6.2/V セットアッププログラムを実行してください.

お使いのネットワークソフトウェアをバージョンアップしたにもかかわらず, 「コン
ピュータがネットワークと接続するよう指定されました」というメッセージが表示さ
れた場合は, [続行]オプションを選択してください. MS-DOS 6.2/V 上で正しくネッ
トワークソフトが動作するはずです.

2　DoubleSpace および Network Drivers
=====================================
ネットワークソフトウェアを DoubleSpace ドライブ上で実行する際に問題が生じた
場合は, CONFIG.SYS ファイル内の DBLSPACE.SYS を読み込むコマンドが, 使用する
ネットワークソフトドライバを読み込むコマンドの前に記述されていることを確認し
てください.

特定のネットワークソフトウェアと DoubleSpace を併用した際に発生した問題につ
いては, 使用するネットワークソフトウェアに関するセクションを参照してください.

3　MS-DOS Shell および Networks
===============================
MS-DOS Shell 内のコマンドプロンプトでネットワークソフトウェアを起動すると, 
ほとんどの場合正常に作動しません. ネットワークソフトウェアを起動するのは, 
MS-DOS Shell を起動する前にしてください.

4　3+Share
==========
3+Share の 1.6 以前のバージョンを使用する場合, すでに MS-DOS 5.0 へのバージョ
ンアップがすんでいれば, 最新のバージョンの 3+Share リダイレクタは MS-DOS
6.2/V 上で作動します. MS-DOS 6.2/V を組み込んでから, コマンドプロンプトで次
のように入力して[Enter]キーを押し, コンピュータを再起動します.

   SETVER MSREDIR.EXE 5.0

MS-DOS 5.0 にリダイレクタが付属していない場合は, Supplemental Disk を注文す
るか, Microsoft Product Support Download Service(MSDL)からディスクをダウンロ
ードしてください.

リダイレクタを入手したら, Supplemental Disk 内の NET.TXT に示した操作方
法に従ってください.

5　Arcnet システム
==================
Arcnet ハードウェアをネットワークにお使いの場合, Microsoft Diagnostics(MSD)
を, MSD /I コマンドを使って実行しなければならないことがあります. /I スイッチ
を使用しなければならない場合は, MSD の Networks オプションは選択しないでくだ
さい.

6　Artisoft LANtastic
=====================
どのバージョンの Artisoft LANtastic をお使いでも, まず LANtastic のサーバー
を休止してから MS-DOS 6.2/V のセットアッププログラムを実行してください. ただ
し, リダイレクタを休止する必要はありません.

Artisoft LANtastic の 3.02 以上のバージョンは, MS-DOS 6.2/V と互換性がありま
す.

LANtastic の 2.5 以前のバージョンをお使いの場合は, メーカーに問い合わせてバ
ージョンアップしてください.

LANtastic の 2.5 から 3.01 までのバージョンをお使いの場合は, 次の操作を行っ
てください.

1) LANtastic のサーバーを休止します. リダイレクタはそのまま実行しておきます.

2) MS-DOS 6.2/V をセットアップします. 

   ネットワーク起動時に,「DOS のバージョンが違います」というメッセージが表示
   されても無視してください.

3) コマンド プロンプトで次のように入力して[Enter]キーを押します.

   SETVER SERVER.EXE 4.00

4) コンピュータを再起動します.

EMM386 をお使いの場合は, ネットワークアダプタカードの使用しているメモリ領域
を除外する必要があります. それには, CONFIG.SYS ファイル内の, EMM386.EXE 用の
DEVICE コマンドに X スイッチを追加します. LANtastic 2Mbps アダプタの既定の設
定を使う場合は, EMM386.EXE 用の DEVICE コマンドに X=D800-DFFF スイッチを追加
してください. メモリ領域を除外しておかないと, EMS メモリを使用するプログラム
を実行する際に問題が発生することがあります.

注意
Microsoft Defragmenter は, サーバーサービスが読み込まれている場合は
LANtastic のネットワークソフトウェアとの互換性がありません. リダイレクタのみ
が読み込まれている場合は, Microsoft Defragmenter は LANtastic のネットワーク
ソフトウェアとの互換性があります.

注意
DoubleSpace を使って LANtastic サーバーのドライブを圧縮することはできません.
ローカルドライバはすべてネットワークドライブと見なされるからです. ドライブを
圧縮するには, サーバーサービスを無効にします. それには, AUTOEXEC.BAT ファイ
ル内のサーバーを読み込むコマンドの前に REM コマンドを追加し, コンピュータを
再起動します. その後 DoubleSpace を使ってドライブを圧縮してください. 
DoubleSpace を実行し終ったら, AUTOEXEC.BAT から REM コマンドを削除してくださ
い.

サーバー上のドライブを圧縮すると, DoubleSpace の機能のうち正常に作動しなくな
るものがでます. LIST, CHKDSK, MOUNT, UNMOUNT などがその機能です. ただし, サ
ーバーサービスを中止すれば直ります.

さらに, NET コマンドもサーバーのドライブ設定を正しく表示しなくなります. 例え
ば,「DoubleSpace に使用できる」はずのドライブ名が表示されません. また, ドラ
イブ C は, 圧縮すると, 誤って物理ドライブと表示されます. ホストドライブが
「DoubleSpace に使用できる」ドライブの後にドライブ名を用いる場合は, それをア
クセス不可能にすることができます. それには, 別のサーバーに接続する際にそのド
ライブ名を使用します.

7　AT&T StarGroup (non-LAN Manager)
===================================
お使いのネットワークソフトウェアが AT&T StarGroup の場合は, MS-DOS 6.2/V セッ
トアップ プログラムを実行してください. セットアップ中「コンピュータがネット
ワークに接続するよう指定されています」というメッセージが表示された場合は,
[続行]オプションを選択し, 画面の指示に従ってください. セットアップ完了後にネッ
トワークソフトウェアを実行できない場合は, 次の操作を行ってください.

1　コマンド プロンプトで次のコマンドを入力してそれぞれ[Enter]キーを押します.

   SETVER SETUP.EXE 4.00
   SETVER ATTSTART.EXE 4.00

2) コンピュータを再起動します.

3) StarGroup のインストール ディスクを使って, 現存のネットワークソフトウェア
   をコンピュータから削除します.

4) StarGroup を再度組み込みます. StarGroup の組み込み中に, 使用中の MS-DOS
   のバージョン番号を尋ねてきたら, MS-DOS 4.0 を選択します.

8　Banyan VINES
===============
Banyan VINES Version 4.0 以上のものは, MS-DOS 6.2/V と互換性があります.
Banyan VINES Version 4.0x を MS-DOS 6.2/V と併用することができますが, その場
合は, ハードディスク上の BANYAN ディレクトリ内に REDIRALL.EXE あるいは
REDIR4.EXE がなければなりません.

これらのファイルがハードディスク上になくても, どちらかのファイルが VINES の
インストールディスクにある場合は, そのファイルを, ハードディスク上の, ネット
ワークファイルの入っているディレクトリにコピーしてください.

ハードディスク上にも, VINES のインストール用ディスクにも, これらのファイルが
2つともない場合は, メーカーにお問い合わせください.

お持ちの VINES が Version 4.0 以前の場合は, メーカーに問い合わせてバージョン
アップしてください.

注意
Banyan VINES は, MemMaker とは互換性がない場合があります. たとえば, お使いの
ネットワークソフトウェアが, 複数の構成要素をそれぞれ異なった UMB 領域に読み
込むような場合, MemMaker が正常に作動しないことがあります.

9　DEC Pathworks および PCSA
============================
お使いの DEC PATHWORKS が DOS Version 4.1 用の場合は, 次の操作を行ってくださ
い.

1) Version 3.3 以前の MS-DOS をバージョンアップする場合は, リダイレクタファ
   イルの REDIR.EXE を更新してください. PATHWORKS のディスクか, あるいはシス
   テムサービスから REDIR.400 を入手し, ファイル名を REDIR.EXE に変更し,
   DECNET ディレクトリへコピーします.

2) MS-DOS 6.2/V を組み込みます. 

DOS 用の PATHWORKS Version 4.0 は, MS-DOS 6.2/V と互換性があります. ただし,
互換性を保つには, 次の操作を行う必要があります.

1) Version 3.3 以前の MS-DOS をバージョンアップする場合は, リダイレクタファ
   イルの REDIR.EXE を更新してください. PATHWORKS のディスクか, あるいはシス
   テムサービスから REDIR.400 を入手し, ファイル名を REDIR.EXE に変更し,
   DECNET ディレクトリへコピーします.

2) MS-DOS 6.2/V を組み込みます.

3) コンピュータを再起動します. ただし, ネットワーク起動用の STARTNET.BAT は
   この時点ではまだ実行しないでください.

4) コマンド プロンプトで次のコマンドを入力してそれぞれ[Enter]キーを押します.

   SETVER USE.EXE 4.00
   SETVER MEMMAN.EXE 4.00
   SETVER LAD.EXE 4.00
   SETVER SHOW.EXE 4.00
   SETVER NETSETUP.EXE 4.00

5) コンピュータを再起動し, STARTNET.BAT を実行します.

注意
PATHWORKS 4.0 を PATHWORKS 4.1 にバージョンアップする場合は, この操作法の中
で入力した SETVER コマンドを禁止しなければなりません. そのためには,
PATHWORKS 4.1 を組み込んだ後で上記のコマンドを入力する際に, 各コマンドの
4.00 の代わりに /D と入力してください.

お使いの PATHWORKS および PCSA が Version 4.0 以前の場合は, メーカーに問い合
わせて更新してください. また, お使いの PATHWORKS および PCSA が Version 3.x 
の場合は, 次の操作を行えば, MS-DOS 6.2/V を使用できます.

注意
MS-DOS Task Swapper 用のサポートは, Version 4.1 以降のDOS 用 PATHWORKS で使
用できます.

一時的に PATHWORKS および PCSA のVersion 3.x をアップデイトするには, 次の手
順に従います.

1) Version 3.3 以前の MS-DOS をバージョンアップする場合は, リダイレクタファ
   イルの REDIR.EXE を更新してください. PATHWORKS のディスクか, あるいはシス
   テムサービスから REDIR.400 を入手し, ファイル名を REDIR.EXE に変更し,
   DECNET ディレクトリへコピーします.

2) MS-DOS 6.2/V を組み込みます.

3) コマンド プロンプトで次のコマンドを入力してそれぞれ[Enter]キーを押します.

   SETVER USE.EXE 4.00
   SETVER EMSLOAD.EXE 4.00
   SETVER SCH.EXE 4.00
   SETVER NETSETUP.EXE 4.00
   SETVER NCP.EXE 4.00
   SETVER DNNETHAT.EXE 4.00
   SETVER LAD.EXE 4.00
   SETVER DLL.EXE 4.00
   SETVER MEMMAN.EXE 4.00
   SETVER DNNETHLD.EXE 4.00
   SETVER SHOW.EXE 4.00

4) コンピュータを再起動します.

10　IBM PC LAN
==============
IBM PC LAN 1.34 をお使いで, レコードロックおよびファイルロックを使用するアプ
リケーションに問題が発生した場合は ICO 3039 付属の, 更新された REDIR50.EXE 
が必要なことがあります. この場合は, メーカーにお問い合わせください.

Version 1.33 以前の IBM PC LAN Program
--------------------------------------
MS-DOS 6.2/V と IBM PC LAN Program (PCLP) を併用するには, 次のどちらかが必要
です.

o IBM PCLP Version 1.34 が組み込まれていること.

o IBM PCLP Version 1.3x で, 1.34 に更新したもの. 更新するためには, 適切な
  IBM Corrective Service Disks を使用します. メーカーに問い合わせて, 必要な
  ソフトウェアを入手してください.

Version 1.30 以前の PCLP の場合は, メーカーに問い合わせて, 必要なソフトウェ
アおよびその操作法を入手してください.

IBM PCLP Version 1.30 から 1.34 までの更新方法は次の通りです.

1) バージョンアップするものが 1.34 の場合は, 手順 3 へ進んでください. そのほ
   かのバージョンの場合は, 必要な IBM PCLP ソフトウェアを組み込みます. それ
   には,添付されている Corrective Service Disk に記述されている操作方法を参
   照してください.

2) コンピュータを再起動し, ネットワークが起動することを確認します.

   このときネットワークが起動しない場合は, 操作を中止し, メーカーに処理方法
   を問い合わせてください.

3) MS-DOS 6.2/V をセットアップします.

   セットアッププログラムでコンピュータが再起動すると, 次のメッセージが表示
   されます.

   PC LAN Program 1.3 を起動しますか (Y/N)？

4) [N]キーを押してから, [Enter]キーを押してください.

5) コマンド プロンプトで次のように入力して[Enter]キーを押します.

   SETVER NET.COM /D

6) コンピュータを再起動します.

   次のメッセージが表示されます.

   PC LAN Program 1.3 を起動しますか (Y/N)？

7) [Y]キーを押してから, [Enter]キーを押します.

   注意
   CONFIG.SYS ファイル内に, MS-DOS をセットアップした後で, IFSFUNC.EXE を読
   み込むコマンドがある場合は, そのコマンドを無効にしてください. それには,
   コマンドの前に REM コマンドを追加します. そしてファイルを保存し, コンピュ
   ータを再起動します.

   注意
   Microsoft Defragmenter は, PCLP サーバー上では実行できません.

11　IBM DOS LAN Requester 1.30 およびそれ以前のバージョン
=========================================================
MS-DOS 6.2/V と IBM DOS LAN Requester を併用するには, 次のどちらかが必要です.

o DOS LAN Requester のファイル. OS/2 LAN Server 1.30.1 に付属のもの.

o DOS LAN Requester のファイル. OS/2 LAN Server 1.30 付属のもので,
  IBM Corrective Service Diskette(CSD)5015 をすでに作動させてあるもの. メー
  カーに問い合わせて必要なソフトウェアを入手してください.

Version 1.30 以前の DOS LAN Requester を実行する場合は, メーカーに問い合わせ
て, 必要なソフトウェアとその操作方法を入手してください.

DOS LAN Requester 1.30 をバージョンアップするには, 次の手順に従います.

1) DOS LAN Requester ソフトウェアのバージョンを, 1.30.1 にバージョンアップし
   ます. それには, IBM Corrective Service Disk 5015 に記述されている操作方法
   に従います.

2) コンピュータを再起動し, ネットワークが起動することを確認します.

   このときネットワークが起動しない場合は, 操作を中止し, メーカーに処理方法
   を問い合わせてください.

3) MS-DOS 6.2/V をセットアップします.

   セットアッププログラムでコンピュータが再起動すると, 次のメッセージが表示
   されます.

   DOS LAN Program 1.3 を起動しますか (Y/N)？

4) [N]キーを押してから, [Enter]キーを押します.

5) コマンドプロンプトで次のように入力して[Enter]キーを押します.

   SETVER NET.COM /D

6) コンピュータを再起動します.

   次のメッセージが表示されます.

   DOS LAN Program 1.3 を起動しますか (Y/N)？

7) [Y]キーを押してから, [Enter]キーを押します.

   注意
   CONFIG.SYS ファイル内に, MS-DOS をセットアップした後で IFSFUNC.EXE を読み
   込むコマンドがある場合は, そのコマンドを無効にしてください. それには, コ
   マンドの前に REM コマンドを追加します. そしてファイルを保存し, コンピュー
   タを再起動します.

12　Net/One PC
==============
Ungermann-Bass Net/One PC が MS-DOS 6.2/V と同時に読み込まれない場合は, 次の
いずれかを行ってください.

o ネットワークソフトウェアのメーカーに問い合わせて, 更新した LOADNIU.EXE を
  入手する.

o MSNET.INI 内の LOADNIU と XNSBIOS の間に UBPAUSE コマンドを追加する.

o バッチファイルからネットワークコマンドを実行する. その際バッチファイル内の
  LOADNIEU と XNSBIOS の間に PAUSE コマンドを追加しておく.

13　Novell Networks
===================
NETWORKJ.TXT ファイル発行時には, ワークステーションシェルの最新バージョンは 
3.31 です. したがって, MS-DOS 6.2/V と併用する際には, 3.31 以降のバージョン
を入手してください.

Version 2.1 以前の NetWare をお使いの場合, もしくはお使いのネットワークソフ
トウェアが次に挙げるファイルを使用する場合は, ネットワークソフトのメーカーに
問い合わせてください.

NET3.COM
NET4.COM
XMSNET3.EXE
XMSNET4.EXE
EMSNET3.EXE
EMSNET4.EXE

Novell ネットワークの実行中に, MS-DOS Shell 内で タスクスワッパを使用できな
い場合は, ネットワークソフトのメーカーに問い合わせてください.

MS-DOS 6.2/V をリモートブートワークステーション上に読み込む事ができない場合
は, メーカーに問い合わせて, RPLFIX.COM を入手してください.

MS-DOS Shell から, あるいは MS-DOS Shell 実行中に LOGIN.EXE, MAP.EXE, 
SESSION.EXE を実行する場合は, 既存のパスに検索パスは追加されません. これらの
プログラムを実行してから MS-DOS Shell を起動してください.

MS-DOS 6.2/V を起動した後で, Novell ログインプログラムを使用して COMSPEC 変
数を設定する場合, COMMAND.COM を再び読み込むことができないときは, CONFIG.SYS
ファイル内の SHELL コマンドを書き換えてください. COMMAND.COM へのパスがルー
トディレクトリを参照していることを確かめ, パスの後ろに C:\DOS\ パラメータが
ある場合は, それを削除してください.

たとえば, 次のような SHELL コマンドラインがあるとします.

   SHELL=C:\DOS\COMMAND.COM C:\DOS\ /P /E:512

これを次のように書き換えてください.

   SHELL=C:\COMMAND.COM /P /E:512

あるいは, メーカーに問い合わせて新しいログイン ファイルを入手してください.

Novell ネットワーク上で Microsoft Undelete を使っているときに問題が発生した
場合は, MAP ROOT コマンドを“削除セントリ”レベルと共に使用します.
(Version 2.15c 以前の Novell Netware をお使いの際は, メーカーに問い合わせて
最新の MAP.EXE を入手してください). たとえば, 次のような書式で MAP ROOT コマ
ンドを使用します (通常はサーバーのログインスクリプトに指定されています).

MAP ROOT F:=COUNT/SYS:USERS/JOEB

この例では, F はネットワークのドライブ名, COUNT はファイル サーバー名, SYS 
はボリューム名, USERS はボリューム SYS 内のサブディレクトリ, そして JOEB は
ネットワーク ユーザーのためのルートになります. JOEB のコンピュータでは,
AUTOEXEC.BAT ファイルには次の書式が含まれます.

UNDELETE /SF

/S スイッチは,“削除セントリ”レベルを指定し, /F スイッチは, 保護するネット
ワークドライブを指定します. AUTOEXEC.BAT ファイルには, 保護する各ドライブに
/S[DRIVE] を指定しておく必要があります.

注意
MS-DOS 用または Windows 用の Microsoft Backup を使ってネットワークへファイル
をバックアップするとき, ネットワークソフトウェアに NetWare 286 を使用してい
る場合は, 最初はガス ゲージの動き方が非常に遅いはずです. これは, NetWare 286
が, 文字ファイルをネットワークへ処理する際の方法が原因です.

注意
ログインスクリプトで使われた %OS_VERSION パラメータは v5.00 を返します. ただ
しこれは, 3.31 までのシェルを使用している場合です. メーカーから更新したシェ
ルがリリースされるまでは, ユーザーのログインスクリプト内の現存の v5.00 ドラ
イブ上では, %OS_VERSION パラメータを使ったり, v6.00 ドライブをマップしないで
ください.

MS-DOS セットアップ プログラムと, 共有の Windows のシステム構成
---------------------------------------------------------------
検索ドライブを使って, ネットワークディレクトリ上の Windows 用初期化ファイル
(PROGRAM.INI, SYSTEM.INI, WINFILE.INI) にアクセスする場合, MS-DOS 6.2/V 付属
の Windows のプログラムを組み込むには, 次の操作を行ってからセットアッププ
ログラムを実行してください.

注意
次の操作方法は, 通常通りセットアッププログラムを実行する場合にも, /E スイッ
チを併用して実行する場合にも, あてはまります.

* コマンドプロンプトで MAP コマンドを使い, 一時的に検索ドライブを無効にしま
す. たとえば, お使いの Windows ディレクトリに対するネットワークドライブの割
当が W の場合, パスが \USER\JOEB であるなら, コマンドプロンプトで MAP DEL W:
と入力して [Enter]キーを押し, 一時的にサーチドライブを削除します. そして, コ
マンドプロンプトで MAP W:=SYS:USER\JOEB と入力して[Enter]キーを押します. そ
れからセットアッププログラムを実行します. セットアップが完了してコンピュータ
が再起動したら, システムは通常の設定に戻ります.

14　Novell Networks および DoubleSpace
======================================
ネットワークソフトウェアに NetWare をお使いの場合は, 必ず NetWare を実行した
後で DoubleSpace を実行してください.

NetWare Lite
------------
NetWare Lite は, ネットワークソフトウェアがリモートファイルシステムを取り付
ける場合,「使用可能な DoubleSpace」ドライブをすべて使用します. その結果, ド
ライブへアクセスしようとすると,「準備ができていません」というエラー メッセー
ジが表示され, Windows のファイルマネージャにはそのドライブがフロッピードライ
ブとして表示されます. さらに, 別のドライブを圧縮しようとしても, DoubleSpace 
はそれまで「使用可能な DoubleSpace」であったドライブを使用することができませ
ん. これは, NET MAP DEL コマンドを使っても同じです.

ドライブの衝突を避けるために, DoubleSpace を カスタムモードで実行してくださ
い. そして, ホストドライブ名を指定するのに, お使いのネットワークソフトウェア
が使用していないドライブ名を使用します. たとえば, ネットワークドライブが現在
F の場合, DoubleSpace を組み込むにはホストドライブ名に E を使用します.

15　PC-NFS
==========
SETVER コマンドを使って, MS-DOS の異なるバージョンをプログラムに伝えている場
合, PC-NFS 3.x は誤ったバージョンを伝えることがあります. このような場合は, 
メーカーに問い合わせてネットワークソフトウェアをバージョンアップしてください.

PC-NFS 3.0 と MS-DOS Shell を併用している場合, MS-DOS Shell は, 現在実際に有
効な状態にないドライブまで表示することがあります. これを避けるには, PC-NFS /D
スイッチを使って, 使用するドライブ数を指定してください. 詳しくは, PC-NFS の
マニュアルを参照してください.

COPY および XCOPY コマンドを PC-NFS 3.01 上で使用できない場合は, メーカーに
問い合わせて PCNFS.SYS デバイスドライバをバージョンアップしてください.

PC-NFS ネットワークソフトウェアをお使いの場合は, 次の操作を行ってから
MemMaker を実行してください.

1) テキストエディタを使って MEMMAKER.INF を開きます. このファイルは, MS-DOS
   ファイルのあるディレクトリにあります.

2) ファイルに次のコマンドラインを追加します.

   *NET

3) ファイルを保存し, MemMaker を実行します.

16　TCS 10Net および DCA 10Net
==============================
DCA 10Net Plus 4.20 (20) 以降, DCA 10Net 3.3 (41) 以降は, MS-DOS 6.2/V と互
換性があります. これ以前のバージョンをお使いの際は, メーカーに問い合わせて更
新してください.

10Net Network Software の, 互換性のあるバージョンを実行する際は, ネットワー
クソフトウェアを無効にしてから MS-DOS 6.2/V セットアッププログラムを実行して
ください. MS-DOS 6.2/V を組み込んだ後で, ネットワークソフトウェアを再び有効
にします.

注意
Microsoft Defragmenter は, サーバーサービスが読み込まれている場合は, 
10Net Network Software との互換性がありません. 読み込まれているのがリダイレ
クタのみの場合は, Defragmenter は, 10Net Network Software と互換性を持ちます.

注意
DoubleSpace を使って 10Net サーバー上のドライブを圧縮することはできません. 
ローカルドライブはすべてネットワークドライブと見なされるからです. ドライブを
圧縮するには, サーバーサービスを無効にします. それには, AUTOEXEC.BAT ファイ
ル内のサーバーを読み込むコマンドの前に, REM コマンドを追加し, コンピュータを
再起動します. それから DoubleSpace を実行してドライブを圧縮します. 
DoubleSpace の実行後に, AUTOEXEC.BAT ファイルから REM コマンドを削除してくだ
さい.

サーバー上のドライブを圧縮すると, DoubleSpace の機能のうち正常に作動しなくな
るものがでてきます. LIST, CHKDSK, MOUNT, UNMOUNT などの機能です. この場合は,
サーバーサービスを中止すれば正常に戻ります.
