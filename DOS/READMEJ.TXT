1  MemMaker, EMM386, およびメモリ管理
=====================================
1.1  インテル製 EMS メモリドライバ (EMM.SYS)
--------------------------------------------
インテル製の EMM.SYS ドライバを Aboveboard と併用する場合は, お使いのシステ
ムが ISA の場合は EMM.SYS  Version 4.0 改訂版 E を, また, お使いのシステムが
MCA その他の場合は EMM.SYS  Version 4.0 改訂版 D を使用してください. 

1.2  PC-NFS を搭載したコンピュータで MemMaker を実行する
--------------------------------------------------------
PC-NFS のネットワーク ソフトウェアをお使いの場合は, MemMaker を実行する前に
次の手順に従ってください.

1) テキスト エディタで MEMMAKER.INF ファイルを開きます. このファイルは MS-DOS
   のあるディレクトリに保存されています.

2) 次のコマンドラインを追加します.

   *NET

3) ファイルを保存し, MemMaker を実行します.

1.3  IBM LAN と MemMaker を併用する
-----------------------------------
MemMaker を実行する前に, お使いの IBM LAN サポートドライバである DXMC0MOD.SYS
用に指定したパラメータを調整しなければならないことがあります. CONFIG.SYS ファ
イル内の DXMC0MOD.SYS を読み込むための DEVICE コマンドに, 次のいずれかのコマ
ンドラインを追加してから, MemMaker を実行してください.

o  冒頭の 3つのパラメータを次の例のようにする

   例 : DEVICE=C:\DXMC0MOD.SYS 400000000001,D800,1

o  パラメータなしにする

   例 : DEVICE=C:\DXMC0MOD.SYS

o  冒頭の 3つのパラメータに相当する数のパラメータとカンマを付ける

   例 : DEVICE=C:\DXMC0MOD.SYS 400000000001,,

1.4  MemMaker と Adaptec SCSI デバイス
------------------------------------------
MemMaker は Adaptec SCSI デバイスと互換性があります. ただし, SCSI デバイスを
サポートするのに ASPI4DOS.SYS ドライバを使用する場合, 既定の設定では, このド
ライバは MemMaker による最適化の対象にはなりません. しかし, SCSI デバイスを
起動ドライブとして使用しないかぎりは, このドライバを最適化の対象に含めること
ができます. そのためには, MEMMAKER.INF から ^ASPI4DOS の項目を削除してから
MemMaker を実行し直してください.

1.5  PC Tools RAMBoost を搭載したコンピュータで MemMaker を実行する
-------------------------------------------------------------------
MS-DOS 上では MemMaker か PC Tools RAMBoost のいずれかを使用できます. 両方を
同時に使用することはできません. MemMaker があると PC Tools RAMBoost は読み込
まれなくなりますが, PC Tools RAMBoost を読み込むコマンドが CONFIG.SYS ファイ
ルから削除されるわけではありません. 

1.6  EMM386 によりアプリケーションエラーが検出された場合
--------------------------------------------------------
EMM386 実行中に「EMM386はアプリケーションにエラー#XXを検出しました」というメッ
セージが表示された場合は, プロセッサが EMM386 の例外的なエラーを検出したこと
を示します. 例外的なエラーが起こりやすいのは, 有効でない条件下でアプリケーショ
ンがプロセッサにコマンドを出した場合です. これらのエラーはたいてい特定のプロ
グラムに関連して発生します. どのアプリケーションでも発生しないエラーが起こっ
たときは, デバイスドライバかメモリ常駐プログラムが原因でエラーが発生した可能
性があります.

このようなエラーの発生を避けるには次の手順に従います.

o  どのプログラムが原因かを確認します. それには, そのプログラムが起動してい
   ないときにエラーが起こるかどうか確かめます. デバイスドライバかメモリ常駐
   プログラムに問題があると思われる場合は, コンピュータ起動時にデバイスドラ
   イバおよびメモリ常駐プログラムをバイパスしてみます.

o  EMM386 を無効にします. あるプログラムに原因がある場合は, EMM386 を無効に
   すると, そのアプリケーションがエラーメッセージを発生します. EMM386 を無効
   にすることで, メモリ条件が変わり, エラーが起こらなくなる可能性もあります.

o  デバイスドライバとメモリ常駐プログラムを読み込む順序を変えてみます. ある
   種のエラーは特定のメモリ条件下では起こらないので, この操作によりエラーが
   発生しなくなる可能性があります.

o  エラー#12 が発生するのは, スタックが正しく使われていないことを示します.
   CONFIG.SYS ファイルに次のコマンドを追加してみてください.

   STACKS=18,512

o  エラー#13 が発生する場合は, プログラムが EMM386 なしでプロテクトモードを
   使おうとしている可能性があります. この場合は, そのプログラムの VCPI コン
   プライアントバージョンを入手するか, EMM386 を読み込まずにアプリケーション
   を実行する必要があります.

1.7  既定の設定では MemMaker は上位メモリ領域を積極的にスキャンしない
---------------------------------------------------------------------
MS-DOS 6.2/V の高速セットアップやカスタムセットアップの既定の設定では,
MemMaker は上位メモリ領域を積極的にスキャンしません. [上位メモリ領域を積極的
にスキャンしますか]オプションは[いいえ]に設定されており, MemMaker は上位メモ
リ ブロックのメモリ アドレス C600 から EFFF までを検査します.

プログラムを実行するために利用できるメモリを増やすには, MemMaker を起動して
[カスタム セットアップ]を選択し, [上位メモリ領域を積極的にスキャンしますか]
オプションを[はい]に設定します. (ただし, コンピュータの機種によっては, 上位
メモリブロックをスキャン範囲に含めることにより正常に起動しなくなる可能性もあ
ります).

参考
最後に MemMaker を実行したときに[上位メモリ領域を積極的にスキャンしますか]オ
プションを[はい]に設定しておくと, 次回 MemMaker を実行したときには, このオプ
ションは[はい]のままになっています.

1.8 使用メモリのバイト数が正しく表示されない場合
------------------------------------------------
16MB 以上のメモリを搭載したシステムで, Microsoft Windows を起動しているとき
に MEM コマンドを実行した場合, 使用拡張メモリ(XMS)のバイト数が正しく表示され
ないことがあります. この問題は, Windowsを起動していないときでも, 他社製の XMS
メモリマネージャを使用している場合に発生します.

これは, 16MB 以上のメモリがあって, ROM BIOS が interrupt 15h, function 88h
経由で, メモリが最大 15MB と返してくるときに発生します.

1.9  MEM /C コマンドを実行するとコンピュータが応答しなくなる場合
----------------------------------------------------------------
特定の機種で, MEM /C コマンドを実行してもコンピュータが応答しないことがあり
ます. この問題は, /C を付けたときに発生し, MEM コマンドのみ, あるいは MEM /D
コマンドでは発生しません.

これは他社製の上位メモリブロック(UMB)マネージャやメモリ常駐プログラム(TSR)を
使ったときに発生することがあります.

この場合には, MemMaker を実行して UMB を最適化してください.

1.10 MEMMAKER.STS ファイルが壊れているというエラーメッセージが表示される場合
----------------------------------------------------------------------------
ADAPTEC 社の 1742A SCSI カードをお使いの場合, MemMaker を実行すると,
MEMMAKER.STS ファイルが壊れているというエラーメッセージが表示されることがあ
ります.

これは, MemMaker が ASPIEDOS.SYS デバイスドライバのサイズを測定しようとした
ときに発生します.

この問題を解決するには, MEMMAKER.INF ファイルを編集して, 次の行を追加してく
ださい.

  ^ASPIEDOS

1.11 EMM386 DMA バッファが小さすぎるというメッセージが表示される場合
--------------------------------------------------------------------
QEMM の動作中に MemMaker を実行すると, 次のようなエラーメッセージが表示され
ます.

   EMM386 DMA バッファが小さすぎます. パラメータ D=48 を付け加えて, 再起動し
   てください.

この問題を解決するには, 次の手順に従います.

1) CONFIG.SYS ファイルを編集して QEMM ステートメントを削除し, QEMM を無効に
   します.

2) HIMEM.SYS と EMM386.EXE(パラメータ D=48 を含めて)を読み込む DEVICE コマン
   ドを追加します. たとえば次のようになります.

   DEVICE=C:\DOS\HIMEM.SYS
   DEVICE=C:\DOS\EMM386.EXE D=48

3) コンピュータを再起動して, MemMaker を実行します.

1.12 QuarterDeck Qemm version 7.02 の DOS-Up 機能
-------------------------------------------------
QuarterDeck Qemm version 7.02 の DOS-Up 機能をお使いの場合(この機能は MS-DOS
のデータ構造のいくつかの部分を上位メモリブロックに移動します), MemMaker は
CONFIG.SYS ファイルから次の 2行を削除できません.

   DEVICE=C:\QEMM\DOSDATA.SYS
   DEVICE=C:\QEMM\DOS-UP.SYS

この 2つの行が残った状態でシステムを起動すると, エラーメッセージが表示されま
す.

この問題を解決するには, CONFIG.SYS ファイルを編集して, 上記のコマンドライン
を無効にします.

1.13 AUTOEXEC.BAT からバッチファイルを起動している
--------------------------------------------------
CALL コマンドを使用してバッチファイルを起動している場合には問題ありませんが,
直接起動している場合は MemMaker の 2 回目の起動が行われずに, コマンドプロン
プトにぬけてしまうことがあります. このときは以下の対処を行ってください.

・EDIT コマンドを使用して AUTOEXEC.BAT を開きます.
・AUTOEXEC.BAT の最下行に移動して, MemMaker が起動されるようになっているか確
  認します. (起動されていなければ, MemMaker は正常に終了していますので EDIT
  コマンドを終了してください)
・そのバッチファイルを CALL コマンドで起動するようにします. 例えば, TEST.BAT
  というバッチファイルを起動していたすると, 次のように指定します.

  CALL TEST.BAT

・AUTOEXEC.BAT を保存して, EDIT コマンドを終了してください.
・コマンドプロンプトに戻ったら, コンピュータをリセットしてください.
・コンピュータが起動すると, MemMaker の英語の画面が表示されます.
・MemMaker の画面に "Continue or exit?" という問い合わせがありますので, スペ
  ースキーにより "Continue" を選択し, ENTER キーを押すとコンピュータがリセッ
  トされます.
・あとは通常の MemMaker の処理が再開されます.

2  Microsoft(R) Windows(TM) operating system
============================================
2.1  Windows のファイルマネージャで圧縮フロッピーディスクを扱った場合
---------------------------------------------------------------------
Windows のファイルマネージャで, 自動取り付けを設定してある圧縮フロッピーディ
スクを操作すると, エラー メッセージまたは何らかの問題が生じる可能性がありま
す. たとえば, 圧縮フロッピーディスクから非圧縮フロッピーディスクに変更した場
合, 圧縮フロッピーディスクのホストドライブのアイコンが選択されたままになりま
す. このような場合は, [F5]キーを 2回押してください.

3  MS-DOS 6.2/V のハードウェア上の互換性
========================================
3.1  コンパック EXTDISK.SYSTEM ドライバ
---------------------------------------
コンパック EXTDISK.SYS ドライバの Version 3.00 およびそれ以降のバージョンは
MS-DOS 6.2/V との互換性があります. 3.00 以前のバージョンをお使いの場合は, メ
ーカーに問い合わせてバージョンアップしてください.

3.2  Hardcard
-------------
a) Hardcard II

   EMM386.EXE の実行中に Plus Development Hardcard II または Hardcard II XL
   が使えない場合は, 排除スイッチ(x=)を指定し, EMM386 が Hardcardの BIOS ア
   ドレスと矛盾しないようにしてください.

   EMM386 を設定するには, MemMaker で実行することもできます.

b) Hardcard 40 および Passport

   Hardcard 40 または Passport の削除可能ディスクを使っていて, CONFIG.SYS 内
   に PLUSDRV.SYS 用の DEVICE コマンドがある場合は, その DEVICE コマンドを無
   効にするか, 削除してください. それから MS-DOS 6.2/V セットアッププログラ
   ムを実行します. セットアッププログラムが完了したら, PLUSDRV.SYS 用の
   DEVICE コマンドを再び有効にするか, 追加してください. その際, このコマンド
   がファイルの最終行になるようにしてください.

c) Hardcard に DoubleSpace を組み込んだ後, 圧縮ドライブにアクセスできなくなっ
   た場合は, 次のいずれかの操作を行ってください.

* AUTOEXC.BAT ファイルに DBLSPACE /MOUNT コマンドを追加し, コンピュータ起動
  時に圧縮ボリューム ファイルを Hardcard に取り付けるようにします.

* デバイスドライバ(たとえば ANSI.SYS など)を, 少なくとも 1つは CONFIG.SYS ファ
  イルに必ず読み込まれるようにします. 読み込まれる順序は, ATDOSXL.SYS の前,
  DBLSPACE.SYS の後にします.

4  マイクロソフト社のプログラム
===============================
4.1  Microsoft Anti-Virus
-------------------------
プログラムファイルを浄化する前に, バックアップコピーを作成してあることを確認
してください. プログラムファイルを浄化したためにファイルが破損した場合は, プ
ログラムを組み込み直します. その後再び Anti-Virus にウィルスが検出された場合
は, 検出自体が誤っている可能性があります. そのようなときは, そのプログラムの
メーカーに, 新しいバージョンのプログラムが入手可能かどうか問い合わせてくださ
い.

MS-DOS 用の Anti-Virus が使用するシグナチャは VIRSIGS.MS ファイルに保存され
ています. また, Windows 用の Anti-Virus が使用するシグナチャは MWAVSCAN.DLL
ファイルに保存されています. 

4.2  Microsoft Anti-Virus がスキャン中に終了してしまう場合
----------------------------------------------------------
MS-DOS 用の Anti-Virus, あるいは Windows 用の Anti-Virus がウィルスをスキャ
ン中, ディスクエラーに遭遇すると, 強制的に終了してしまい, スキャンを続行しな
いことがあります.

この場合には, ScanDisk を実行してから, Anti-Virusを実行し直してください.

4.3  TI4000 または Gateway NOMAD コンピュータで Microsoft Backup を実行する
---------------------------------------------------------------------------
Turbo 機能と, Windows 用および MS-DOS 用の Microsoft Backup とが衝突しないよ
うに, CONFIG.SYS ファイル内の BATTERY.PRO を読み込む DEVICE コマンドに /L0
スイッチを追加します. または, Microsoft Backup を実行する前に, MS-DOS のコマ
ンドプロンプトで SETPOWER /L0 と入力して[Enter]キーを押します.

4.4  Microsoft Defragmenter
---------------------------
Defragmenter の実行時にメモリ不足のメッセージが表示される場合は, MEM コマン
ドを実行して, コンベンショナルメモリ, 上位メモリ, および拡張メモリの大きさを
調整します. Defragmenter は, 実行時にコンベンショナルメモリ全部と, 拡張メモ
リを 384KB, 上位メモリを 12KB 使用します. 

使用可能なメモリを増やすには, 『MS-DOS 6.2/V ユーザーズガイド』の
「7.5.1 MS-DOS プログラムを使用中に“メモリが足りません”というメッセージが
表示される」に示した操作を行ってください.

使用可能な拡張メモリが 384KB 未満しかないときは, 『MS-DOS 6.2/V ユーザーズ
ガイド』の「7.4.2 拡張メモリ(XMS)の解放」に示した操作を行ってください.

4.5  ScanDisk が問題を修復していないにもかかわらず, 修復したと表示する場合
--------------------------------------------------------------------------
ScanDisk が MDBPB(Microsoft DoubleSpace BIOS パラメータブロック)内で, 何度も
破損を検出し, 実際には問題を修復していないにもかかわらず, 修復したと表示して
くることがあります.

これは, 最小評価圧縮率(ECR)の計算中に ScanDisk が問題のある箇所を避けること
によって, 発生します.

この問題を解決するには, DoubleSpace の圧縮率を設定し直してから, ScanDisk を
実行してください. たとえば, 次の操作を行います.

1) DoubleSpace の圧縮率をある固定値に設定します(つまり, DoubleSpace の圧縮率
   を既定の設定以外のものにします). たとえば, MS-DOS のコマンドプロンプトで
   DBLSPACE /RATIO=2 と入力して[Enter]キーを押します. 圧縮率がすでに 2:1 に
   なっていたら, 3:1 に設定します.

2) ScanDisk を実行して, MDBPB の問題のある箇所を修復します.

3) DoubleSpace を実行して, 圧縮率を設定し直します. 詳しくは, MS-DOS のコマン
   ドプロンプトで DBLSPACE /RATIO と入力して[Enter]キーを押します.

4.6  Windows が起動しているときでも ScanDisk を実行できる場合
-------------------------------------------------------------
ScanDisk は, Windows が起動しているときは /CHECKONLY モードでのみ動作します.
しかし, まれに, Windows 起動しているときに SCANDISK /CHECKONLY /SURFACE と入
力して[Enter]キーを押すことで, クラスタスキャンを実行することができることが
あります.

4.7  Undelete で 0 Byte のファイルを復元できない場合
----------------------------------------------------
Undelete の保護レベルが“削除追跡”に設定されていると, 0 Byte のファイルは復
元できません.

この場合は, UNDELETE /DOS コマンドを使って, ファイルを復元することができます.

1) 削除されたファイルを含むディレクトリに変更します.

2) MS-DOS のコマンドプロンプトで, 次のように入力して[Enter]キーを押します.

   UNDELETE *.* /DOS

アプリケーション(Lotus 1-2-3 のように)が 0 Byte のファイルを使用していて, そ
のファイルを復元することができない場合は, メーカーに問い合わせて同じファイル
を入手してください.

4.8  1024FD
-----------
1024FD は，MS-DOS ファイルベースでの互換性を維持するためのユーティリティです.
そのためディスクの物理セクタを参照する一部のコマンド, ユーティリティは動作し
ません. 現在動作できる環境は, 5.25インチ/2HD ドライブの コンパック社の
DESKPRO シリーズおよび Prolinear シリーズの 3 モード対応機種です.

MS-DOS 6.2/V に付属するコマンドの動作

5.25 インチの 1024BPS フロッピーディスクでは DISKCOMP コマンドを実行できませ
ん.
3.5 インチの 1024BPS フロッピーディスクでは ScanDisk コマンドを実行できませ
ん.

コンパック社の DESKPRO シリーズの 3 モードフロッピーディスク

コンパック社の DESKPRO シリーズでは, 1024FD を組み込んだ状態で, 1024BPS のフ
ロッピーディスクを挿入直後に, DIR コマンド等を実行するとエラーが発生する場合
があります. その場合には，1024FD を組み込む際に /T オプションを追加してくだ
さい. なお，このオプションスイッチは, 1024FD を組み込む際のみ有効となります.
また，デバイスドライバとして実行した場合にも，このスイッチは使用できます.

Windows エンハンスドモードと MS-DOS Shell のタスクスワップがオンの時

1024FD は，Windows のエンハンスドモードと MS-DOS Shell の タスクスワップに対
応しています. その状態では各タスク内で Enable/Disable が可能ですが，処理は独
立しています. Windows を起動し, MS-DOS プロンプトで 1024FD を組み込んだあと,
ファイルマネージャで 1024BPS のメディアにアクセスしても, エラーとなります.
1024FD は MS-DOS プロンプトでのみ有効となっているからです,  
また，FORMAT 時や DISKCOPY等, 物理セクタを直接参照している最中にはタスク切り
替えを行わないでください. タスクを切り替えた場合には, フロッピーディスクを破
損する場合があります. また，SHARE コマンドを必ず組み込んでください.

メディアディスクリプタ

1024FD を通じて, 1024BPS のメディアディスクリプタは変更できません. また，メ
ディアディスクリプタは 3.5インチの 1024BPS のフロッピーディスクでは，F0H に
5.25インチでは，F9H に変換されます.

ブートセクタについて

1024FD を通じて 1024BPS のフロッピーディスクのブートセクタは, 最初の 512 byte
しかアクセスできません.

フロッピーディスクモデルでの動作

ハードディスクが装着されていない機種では, SmartDrv の 後に 1024FD は組み込め
ません. 組み込む場合には, CONFIG.SYS ファイルか AUTOEXEC.BAT ファイルの
SmartDrv の前に 1024FD を組み込むようにしてください.

4.9 Vsafe
---------
Vsafe は 画面モード 03H,73H 以外は対応していません. V-TEXT の画面モード 70H,
71H では起動できません. また, 起動後, 画面表示もされません. この時, ユーザーが
指定した警告が発生した場合, 通常(画面モード 03H,73H)は警告メッセージが表示され,
ユーザーの指示を仰ぎますが, 画面表示ができないため Vsafe ではすべて「中止」の
オペレーションが行われたことになります.
従って, アプリケーションなどではエラーが発生したことになり処理が中断する可能性
があります. また, グラフィックスモードでも同様で, グラフィックスモードで動作す
るアプリケーション上で警告が発生した場合もエラーを検出します.
Vsafe は, Smartdrv.exe との併用はなるべく避けてください. 書き込みなどの警告が
発生した場合, 警告メッセージで<中止>を選択しても Smartdrv がエラーを検出してエ
ラーメッセージを表示します.
この場合, Smartdrv では中断処理がないため再試行をするしかありません. Vsafe を
使用する場合は Smartdrv のキャッシュ書き込みを無効にしてください.

4.10  CodeView
--------------
お使いのシステムが 80386 メモリマネージャ(EMM386.EXE など)と, 拡張メモリを使
用するデバイスドライバまたはプログラムを使用している場合, CodeView CV.EXE の
3.0 から 3.13 までのバージョンを使用するとデータが失われることがあります. バ
ージョン番号を調べるには, MS-DOS のコマンドプロンプトで CV.EXE と入力して
[Enter]キーを押します.

4.11 FONTEDIT 使用時の注意事項
------------------------------
外字フォントエディタの漢字参照または外字登録, 参照画面では, JIS コードで参照
できないエリアでは選択メニューに[J:コード]メニューが表示されません.

5  他社のプログラム
===================
5.1  4DOS および NDOS
---------------------
4DOS と NDOS は MS-DOS 6.2/V との互換性があります. ただし, MS-D0S 6.2/V の新
機能のいくつか(MemMaker, LOADHIGH /L スイッチ, DIR コマンドの /C スイッチ,
起動コマンドのバイパスなど)を使用するには, それぞれのメーカーに問い合わせて
4DOS 4.02 以降および Norton Utilities 7.0 以降のバージョンを入手してください.

複数のシステム構成を設定する場合, 4DOS または NDOS は, 自動的に AUTOEXEC.BAT
ファイルを実行することはありません. 自動的に実行させるためには, CONFIG.SYS
ファイルの SHELL コマンドラインに /P スイッチを追加してください.

4DOS Help または NDOS Help の代わりに MS-DOS 6.2/V Help を使用するには, ヘル
プを起動するときに COMMAND /C HELP と入力するか, 4DOS または NDOS のエイリア
スを定義して MS-DOS 6.2/V のディレクトリから HELP.COM を実行してください.

5.2  Above Board 286 および Above Board Plus Installation プログラム
--------------------------------------------------------------------
1989年 5月以前の日付の Above Board Installation プログラムを使用するには, 使
用する前に必ず SMARTDrive や RAMDrive のような, 拡張メモリを使うプログラムを
無効にしてください. これらのプログラムを有効なままにしておくと, データが失わ
れることがあります. Above Board を組み込んだ後で, これらのプログラムを再び有
効にしてください.

5.3  Colorado Tape Backup
-------------------------
2種類の VFINTD.386 が読み込まれていますというメッセージが表示された場合は,
SYSTEM.INI を編集する必要があります. 次の手順に従ってください.

1) SYSTEM.INI を開き, [386Enh]セクションに移動します. 次のようなコマンドライ
   ンが記述されているはずです.

   DEVICE=C:\TAPE\CMSDTAPE.386
   DEVICE=C\DOS\VFINTD.386

2) Colorado Tape Backup のプログラムを使用する場合は, VFINTD.386 用の DEVICE
   コマンドの前にセミコロン(;)を追加します. また, Microsoft Backup を使用す
   る場合は, CMSDTAPE.386 用の DEVICE コマンドの前にセミコロン(;)を追加します.

3) ファイルを保存し, Windows を再起動します.

5.4  Fastback Plus
------------------
Fifth Generation Systems Fastback Plus の 3.0 以前のバージョンをお使いの場合
は, Fastback Plus または Fastback Plus のセットアッププログラムを実行する前
に, LOADFIX コマンドを使って, データが失われないようにします. それには,
MS-DOS のコマンドプロンプトで次のように入力して[Enter]キーを押します.

LOADFIX FB.EXE

または

LOADFIX FBINSTAL.EXE

5.5  Norton Desktop for Windows 2.0
-----------------------------------
セットアッププログラムにより[ツール]メニューが追加されます. このメニューには,
[バックアップ]コマンド, [アンチウィルス]コマンド(ただし, これらの Windows 用
プログラムを組み込まれている場合), および[DoubleSpace 情報]コマンド
(DoubleSpace が組み込まれている場合)が含まれます.

SmartCan が組み込まれているドライブを DoubleSpace で圧縮すると, 以後, システ
ムに問題が生じる可能性があります. これを避けるには, Norton Desktop for
Windows を組み込み直してください.

Norton Anti-Virus および Norton DoubleSpace についての詳細は, 次のセクション
を参照してください.

5.6  Norton Utilities
---------------------
Norton Speed Disk(Version 7.0 以前)の[Clear Space]オプションは, DoubleSpace
ドライブと互換性がありません. Norton Speed Disk の新しいバージョンについては,
メーカーにお問い合わせください.

WipeInfo ユーティリティは, DoubleSpace ドライブでは使用しないでください. ク
ラスタが失われることがあります. このオプションをすでに使用している場合は,
SCANDISK コマンドを使って破損クラスタを修復してください.

Norton Anti-Virus を実行中に, NAV_.SYS ファイルのあるドライブを DoubleSpace
で圧縮すると, NAV_.SYS ファイルのコピーがホストドライブに作成されます. これ
により, 圧縮作業中のウィルス感染を検出することができなくなります. 圧縮作業が
終了したら, ホストドライブの NAV_.SYS ファイルを安全に削除することができます.

Norton Cache または Norton Speedrive をお使いの場合は, これらのユーティリティ
を DBLSPACE.SYS を読み込むコマンドより先に読み込むようにしてください. 詳しく
は, セクション 6.27 を参照してください.

5.7  PC Tools
-------------
PC Shell にドライブ上のすべてのディレクトリおよびファイルが表示されない場合
は, PC Shell を終了し, メーカーに問い合わせてバージョンアップしてください.

注意
そのドライブでは PC Shell を使用しないでください. 重大なデータの損失が生じる
可能性があります. これは, 圧縮ドライブ, 非圧縮ドライブのどちらにも影響します.

PC Tools の DISKFIX /SCAN オプションを使用すると, DoubleSpace ドライバ内のク
ラスタが破損することがありますので, このオプションは使用しないでください(す
でにこのオプションを使用している場合は, SCANDISK コマンドを使って破損クラス
タを修復してください).

PC Tools の Version 7.1 の FORMAT コマンドは, MS-DOS 6.2/V と衝突します.
MS-DOS 6.2/V の FORMAT コマンドを使用してください.

PC Tools の Version 6.0 および 5.5 の COMPRESS コマンドは, DoubleSpace との
互換性がありません.

5.8  QEMM の Stealth DoubleSpace 機能
-------------------------------------
QEMM 7.01 の Stealth DoubleSpace 機能は MS-DOS 6.2/V とは互換性がありません.
QEMM 7.02 の場合は MS-DOS 6.2/V と互換性がありますが, DEFRAG.EXE や
DoubleSpace の自動取り付け機能とは互換性がありません. QEMM 7.03 は MS-DOS
6.2/V および DoubleSpace と完全な互換性があります. QEMM 7.03 は, QEMM 7 のユ
ーザーであれば, BIX, CompuServe, QuarterDeck BBS などを通じて入手できます.
またはメーカーにお問い合わせください.

QEMM 7.01 または QEMM 7.02 の Stealth DoubleSpace 機能をお使いの場合は, 以下
の説明を参照してください.

QEMM 7.01 の場合
----------------
QEMM 7.01 の Stealth DoubleSpace 機能は MS-DOS 6.2/V とは互換性がありません.
QEMM 7.01 およびその Stealth DoubleSpace 機能を使っていると, コンピュータは
正しく起動しません.

Stealth DoubleSpace 機能を使っているためにコンピュータが正しく起動できないと
きは, コンピュータ起動時に[F8]キーを押します. そして, 次のプロンプト以外のプ
ロンプトが表示されたら[Y]キーを押してください. 

   DEVICE=C:\QEMM\ST-DBL.SYS [Y/N]?

このプロンプトが表示されたら[N]キーを押してください. (ST-DBL.SYS のパス名は
これと違っていることもあります). コンピュータが起動したら, CONFIG.SYS ファイ
ルを以下のように変更してください.

o  REM コマンドを使って, ST-DBL.SYS を読み込むコマンドを無効にします.
o  DBLSPACE.SYS を読み込むコマンドを追加します. たとえば, 次のようなコマンド
   を追加します.

   DEVICE=C:\DOS\DBLSPACE.SYS /MOVE

QEMM 7.02 の場合
----------------
QEMM 7.02 の Stealth DoubleSpace 機能は, MS-DOS 6.2/V と互換性があります. た
だし, 以下の場合は別です.

*  DoubleSpace の自動取り付けがオンになっているときは, QEMM 7.02 の
   Stealth DoubleSpace 機能は使用できません. 自動取り付けをオフにするには,
   MS-DOS のコマンドプロンプトに DBLSPACE /AUTOMOUNT=0 と入力して[Enter]キー
   を押し, コンピュータを再起動します.

*  QEMM 7.02 の Stealth DoubleSpace ドライバが組み込まれているときは,
   Microsoft Defragmenter (DEFRAG.EXE) は実行できません. DEFRAG.EXE を実行す
   るには, Stealth DoubleSpace ドライバを無効にしてコンピュータを再起動しま
   す. DEFRAG.EXE の実行が終了したら, 再び Stealth DoubleSpace ドライバを有
   効にしてコンピュータを再起動してください.

5.9 Johnson Computer Systems PC-Vault および PC-Vault Plus
-----------------------------------------------------------
ハードディスク保護システム PC-Vault または PC-Vault Plus の Version 4.6 以前
をお使いの場合は, DoubleSpace を使う際, [Maximum Floppy Boot Protection] オ
プションを選択しないでください.

Version 4.6 以前の PC-Vault または PC-Vault Plus では,
[Maximum Floppy Boot Protection]オプションは MS-DOS 6.2/V と互換性がなく, デ
ータが失われる可能性があります. Version 4.6 以前の PC-Vault または PC-Vault
Plus をお使いの場合は, メーカーに問い合わせてアップグレードバージョンを入手
できるか確認してください.

5.10 AddStor DoubleTools
------------------------
AddStor DoubleTools Version 1.0 には, MS-DOS 6.2/V とは互換性のないバージョ
ンの DBLSPACE.BIN が含まれています. DoubleTools を組み込むときは,
[Install enhanced DoubleSpace drivers]チェック ボックスをオンにしないでくだ
さい. オンにすると, MS-DOS 6.2/V の DBLSPACE.BIN ファイルが DoubleTools の
DBLSPACE.BIN に置き換えられ, 次にコンピュータを起動するときに「DBLSPACE.BIN
のバージョンが違います」というメッセージが表示されて, 圧縮ドライブを取り付け
ることができなくなります.

5.11 Turtle Beach 社のサウンドカード
------------------------------------
MemMaker を実行すると, Turtle Beach 社のサウンドカードは正常に機能しなくなる
ことがあります.

これは, Turtle Beach 社のサウンドカードの ROM が使用しているメモリ領域を,
MemMaker が検出しないからです.

この問題を解決するには, 次の手順に従います.

1) CONFIG.SYS ファイルを編集してパラメータ X= を追加し, サウンドカードが使用
   しているメモリ領域を除外します. たとえば次のようになります.

   DEVICE=C:\DOS\EMM386.EXE NOEMS HIGHSCAN X=D000-D1FF X=D6000-D7FF

2) MemMaker をカスタム セットアップで実行します.

3) [現在の EMM386 メモリ設定をこのまま保持しますか?]というオプションに対して,
   [はい]を選択します.

5.12 LANtastic サーバー上で Windows 用の Undelete を実行できない場合
--------------------------------------------------------------------
LANtastic サーバー上で Windows 用の Undelete を実行すると, 次のようなエラー
メッセージが表示されます.

   ディスクドライブが見つかりません. 操作を完了できません.

注意
MS-DOS 用の Undelete の場合は, この問題は起こりません.

LANtastic がサーバーモードで実行されているときはすべてのローカルドライブがネッ
トワークドライブと見なされるため, ローカルドライブ上では Windows 用の
Undelete を実行できません. Windows 用の Undelete を実行するには, サーバーサ
ービスを無効にする必要があります. それには, AUTOEXEC.BAT ファイルのサーバー
を読み込むコマンドの先頭に REM コマンドとスペースを追加して, コンピュータを
再起動します.

5.13 $FDD35IA.SYS
-----------------
IBM PS/V などのコンピュータに付属する $FDD35IA.SYS (3 モード FDD デバイスドラ
イバ) を組み込んでいる場合は, DBLSPACE, DEFRAG, Anti-Virus, Microsoft Backup 
などのような全画面を使用するユーティリティは使わないでください.
$FDD35IA.SYS により画面が壊されたり, 不具合が発生することがあります. MS-DOS 
6.2/V の基本的なコマンド (COPY, DEL など)のみご使用ください.

6  DoubleSpace
==============
6.1  XtraDrive ディスク圧縮プログラムを DoubleSpace に置き換える
----------------------------------------------------------------
XtraDrive ディスク圧縮プログラムをお使いの場合は, 付属のセットアッププログラ
ムで圧縮を解き, その後 DoubleSpace を組み込んでください.

6.2  Stacker 2.x または 3.0  ディスク圧縮プログラムを DoubleSpace に変換する
----------------------------------------------------------------------------
Stacker 2.x または 3.0 を使ってドライブを圧縮している場合は, ユーティリティ
を使って, Stacker で圧縮したデータを DoubleSpace のフォーマットに自動的に変
換することができます. このユーティリティを入手するには, パッケージに入ってい
る"各種関連製品のご案内"をお読みください.

Version 2.x および 3.0 以外の Stacker を使ってドライブを圧縮している場合は,
セクション 6.3 および 6.4 に示す操作を実行してください.

6.3  Stacker 3.1 ディスク圧縮プログラムを DoubleSpace に置き換える
------------------------------------------------------------------
Stacker 3.1 を使ってドライブを圧縮している場合は, 次の操作を行って
Stacker 3.1 を削除し DoubleSpace を組み込んでください.

1) Stacker の UNSTACK コマンドを使ってすべての Stacker ドライブの圧縮を解除
   します. (Stacker で圧縮したフロッピーディスクも, Stacker の
   StackerAnywhere 機能で設定されていない場合は, ハードディスク ドライブと同
   じように圧縮解除してください).

2) 起動ハードディスク ドライブのルート ディレクトリに切り替え, 次のコマンド
   を入力してそれぞれ[Enter]キーを押します.

   ATTRIB -R -H -S STACKER.INI
   ATTRIB -R -H -S DBLSPACE.BIN
   DEL STACKER.INI
   DEL DBLSPACE.BIN

3) コンピュータを再起動します.

4) MS-DOS のコマンドプロンプトで DBLSPACE と入力して[Enter]キーを押し,
   DoubleSpace のセットアッププログラムを実行します.

6.4  他のディスク圧縮プログラムを DoubleSpace に置き換える
----------------------------------------------------------
Stacker 2.x, 3.0, 3.1, および XtraDrive 以外のディスク圧縮プログラムを
DoubleSpace に変換するには, 次の手順に従います.

1) MS-DOS 6.2/V を組み込みます. すでに組み込んである場合はその状態でかまいま
   せん.

2) MS-DOS 用の Microsoft Backup で, ハードディスク上のファイルをバックアップ
   します. 

3) セットアップディスクがドライブ A で使用可能であれば,“Disk1-Setup”をドラ
   イブ A に挿入し, コンピュータを再起動します. セットアッププログラムの起動
   画面が表示されたら, [F3]キーを 2回押してセットアッププログラムを終了します.

   セットアップ ディスクがドライブ A で使用可能でない場合は, ドライブ A 用の
   起動ディスクを作成します. まず,“Disk1-Setup”をドライブ B に挿入し, 空の
   フロッピーディスクをドライブ A に挿入します. MS-DOS のコマンドプロンプト
   で B:SETUP /F と入力して[Enter]キーを押します.

   いくつかのオプションが表示されたら, ドライブ A のフロッピーディスクに
   MS-DOS を組み込むオプションを選択します. セットアッププログラムが完了した
   ら, ディスクをドライブ A に入れたままコンピュータを再起動します.

4) FORMAT コマンドを使って, すべての圧縮ファイルを含むファイルのあるドライブ
   をフォーマットします. このファイルの保存場所がわからない場合は, お使いの
   ディスク圧縮プログラムのマニュアルを参照してください.

   ドライブ C をフォーマットし直す場合は, /S スイッチを追加してシステム ファ
   イルも転送してください.

5) ドライブ C のフォーマットが完了したら,“Disk1-Setup”がドライブ A か B に
   入っていることを確認し, MS-DOS のコマンドプロンプトで A:SETUP または
   B:SETUP と入力して[Enter]キーを押します.

以下, 画面の指示に従ってください.

6) セットアップが完了したら, MS-DOS のコマンドプロンプトで DBLSPACE と入力し
   て[Enter]キーを押し, DoubleSpace をセットアップします. 以下, 画面の指示に
   従ってください.

7) MS-DOS 用の Backup を使って, バックアップ ファイルを復元します.

注意
Microsoft Backup を実行すると, もう一度設定し直す必要があります. また, バッ
クアップファイルのあるフロッピーディスクから, カタログファイルも復元する必要
があります. そのためには, [復元]ダイアログ ボックスで[カタログ]ボタンを選択
します.

6.5  DoubleSpace セットアッププログラムが, お使いのディスクキャッシュプログ
     ラムと互換性がない場合
---------------------------------------------------------------------------
DoubleSpace セットアッププログラムが, お使いのディスクキャッシュプログラムと
互換性がない場合は, CONFIG.SYS ファイルまたは AUTOEXEC.BAT ファイルを開き,
ディスクキャッシュプログラムを読み込む DEVICE コマンドを削除してください. ディ
スクキャッシュを使用したい場合は, AUTOEXEC.BAT ファイルに MS-DOS 6.2/V
SMARTDRV プログラム用のコマンドラインを追加します. たとえば, MS-DOS のファイ
ルが DOS というディレクトリにある場合は, 次のようなコマンドラインを追加しま
す.

C:\DOS\SMARTDRV.EXE

テキストエディタを終了し, コンピュータを再起動します. そして DoubleSpace を
もう一度実行します.

6.6  指定したドライブでディスクキャッシュが使用できない場合
-----------------------------------------------------------
MS-DOS 6.2/V の DoubleSpace を使って, DoubleSpace をアンインストールしたあと,
次のようなエラーメッセージが表示されることがあります.

   警告:指定されたドライブでは, ディスク キャッシュを使用することはできません

この問題は, DoubleSpace のホストドライブを特にキャッシュするように SMARTDRV
をカスタマイズしたときに発生します. たとえば, SMARTDRV は次のようになってい
ます.

   SMARTDRV /X H 512 512

ここで, C は DoubleSpace の圧縮ドライブで, H はホスト ドライブです.

DoubleSpace のアンインストール プログラムは, DoubleSpace の圧縮ドライブの圧
縮を解除しても, AUTOEXEC.BAT ファイルの SMARTDRV.EXE コマンドのドライブ名を
調整しません.

この問題を解決するには, AUTOEXEC.BAT ファイルを編集し, 正しいドライブ名を指
定します.

6.7  EXTDISK.SYS が, ドライブ名についての警告を表示した場合
-----------------------------------------------------------
コンパック製コンピュータで DoubleSpace を使用していて, CONFIG.SYS ファイルで
EXTDISK.SYS デバイスドライバを読み込む場合, EXTDISK.SYS が読み込まれるときに
次のメッセージが表示されます.

重要
EXTDISK.SYS は, ドライブ名を割り当てる最初のデバイスドライバではありません.
物理ハードドライブ名は連続しません.

それでも EXTDISK.SYS は正しく作動します. 上記のメッセージが表示されるのは,
EXTDISK.SYS はドライブ名を割り当てる最初のモジュールであるはずが, そうではな
くなっているからです. つまり, DBLSPACE.BIN が CONFIG.SYS の前に読み込まれて
ドライブ名を割り当ててしまうので, EXTDISK.SYS は最初のものではなくなってしま
うのです. (EXTDISK.SYS は, DBLSPACE.SYS デバイスドライバが CONFIG.SYS に読み
込まれていても, 関係なくメッセージを表示します).

6.8 起動ドライブを使用するには特殊なデバイスドライバが必要な場合
-----------------------------------------------------------------
起動用のハードディスク ドライブを使用する際に, CONFIG.SYS ファイル内でデバイ
スドライバを指定する必要がある場合は, そのドライブは圧縮しないでください. 圧
縮してしまうと, コンピュータが正しく起動できません. DoubleSpace が起動ドライ
ブにアクセスすることができないからです. (これは, MS-DOS が DBLSPACE.BIN を読
み込むのが, CONFIG.SYS ファイル内のデバイスドライバの起動よりも前になるから
です. DBLSPACE.BIN は, MS-DOS の一部で圧縮ドライブにアクセスするものです).

特殊なデバイスドライバが必要な起動ドライブのあるコンピュータに DoubleSpace
を組み込むには, DoubleSpace セットアッププログラムを使って, 起動ドライブ以外
のドライブを圧縮するか, 既存のいずれかのドライブの空き領域に新しく圧縮ドライ
ブを作成してください.

6.9 ファイル属性を変更した後に圧縮していないドライブを最適化する
-----------------------------------------------------------------
圧縮したボリューム ファイルの属性を変更しない限り, 圧縮ドライブも非圧縮ドラ
イブも, Microsoft Defragmenter またはその他の最適化プログラムを使って安全に
最適化することができます.

注意
圧縮したボリューム ファイルの属性を変更して, 圧縮ドライブを取り付けずに非圧
縮ドライブを最適化すると, データが失われることがあります.

非圧縮ドライブを完全に最適化するには, まず非圧縮ドライブ上にあるすべての圧縮
ドライブを取り外し, すべての属性を解除してから, Defragmenter またはその他の
最適化プログラムを使用します.


6.10 ドライブ圧縮中に Microsoft Defragmenter のメモリが足りなくなった場合
-------------------------------------------------------------------------
ドライブ圧縮中に Defragmenter のメモリが足りなくなった場合は, DoubleSpace を
終了し, セクション 4.4 に示した操作を行ってください.

その操作を行った後も, Defragmenter のメモリが足りない場合は, ハードディスク
上のファイルが多すぎて Defragmenter にはまとめ切れないことが原因の場合があり
ます. プログラムを正しく動かすには, ファイルをいくつか削除するか, フロッピー
ディスクまたはネットワークドライブへ移動してください.

6.13 DoubleGuard の警告メッセージが表示された場合
-------------------------------------------------
DoubleGuard の警告メッセージが表示されるのは, DoubleSpace が使用しているメモ
リにアプリケーションが損傷を与えてしまったことを, DoubleGuard が検出した場合
です. このような場合, データにこれ以上の損傷を与えないためにコンピュータは停
止します.

通常, 各プログラムは固有のメモリ領域を持っており, 他のプログラムがすでに使用
しているメモリは使用しません. しかし, 中には, 他のプログラムのメモリを誤って
使用してしまうためにプログラムエラーを引き起こすプログラムもあります. このよ
うなプログラムが DoubleSpace のメモリを使用すると, DoubleSpace が保存するデ
ータにそのプログラムのデータが上書きされてしまいます. DoubleSpace がメモリに
保存するデータには通常, 現在使用中のファイルが含まれますから, データが損失す
る可能性があります.

DoubleSpace の DoubleGuard は, 他のプログラムが DoubleSpace のメモリを侵害し
ていないかを検査し, DoubleSpace のメモリが侵害されていた場合には, データ損失
の危険を最小限にするためにコンピュータを停止します. (それでもディスクが動作
し続ける場合は, そのドライブ上のデータの一部または全部が失われる可能性があり
ます. DoubleSpace によりメモリに保存されているデータが, 他のプログラムによっ
て無効なものになっているからです).

DoubleGuard の警告メッセージが表示されたときは, 次の操作を行います.

1) 電源スイッチをオフ, オンして, コンピュータを再起動します.

2) コマンド プロンプトに次のように入力して[Enter]キーを押します.

   SCANDISK /ALL

ScanDisk が起動してすべてのドライブを検査し, DoubleSpace のメモリを侵害した
プログラムによって生じたと思われる問題を修復します.

3) DoubleGuard の警告メッセージが表示されるまでに実行していたプログラム名を
   メモしておいてください. このプログラムが, DoubleGuard に警告メッセージを
   表示させたプログラムである可能性があります. DoubleGuard の他の警告メッセ
   ージも表示されたときは, そのときどのような操作を行なっていたか, どのよう
   な場合に警告メッセージが表示されるかを確認してください.

6.14 圧縮ドライブの分断化が進んでいて取り付けできない場合
---------------------------------------------------------
コンピュータ起動時に ｢圧縮ドライブ X の分断化が進んでいて取り付けできません｣
または ｢X:\DBLSPACE.nnn ファイルは分断化が進んでいて取り付けできません｣
(X はドライブ名を表す) というメッセージが表示された場合は, 圧縮ボリュームファ
イルがハードディスクの分断化された部分に分かれて保存されているために, ドライ
ブが取り付けできないことを意味します. (DoubleSpace はそれぞれの圧縮ドライブ
を, 圧縮ボリュームファイル (CVF) と呼ばれる特殊なファイルに保存します. CVF
は, 隠しファイル, システムファイル, 書き込み禁止ファイルとしての属性を持って
いて, 非圧縮ドライブに保存されます).

これを解決するには, DBLSPACE.INI ファイルの MaxFileFragments の設定値を大き
くします. このエラーメッセージが表示されるのは, 圧縮ボリュームファイルのフラ
グメントの数が MaxFileFragments の設定値より大きい場合だからです.
MaxFileFragments の設定値を大きくするには, 次の操作を行います.

1) 起動ドライブのルートディレクトリに切り替えます. (起動ドライブが圧縮されて
   いる場合は, そのホストドライブに切り替えます).

2) 次のコマンドを入力して[Enter]キーを押します.

   TYPE DBLSAPCE.INI

3) MaxFileFragments の現在の設定値をメモします.

4) DBLSPACE /MAXFILEFRAGMENTS コマンドを使って MaxFileFragments の設定値を大
   きくします. たとえば, 現在の設定値が 128 であるとすると, 次のように入力し
   て [Enter] キーを押します.

   DBLSAPCE /MAXFILEFRAGMENTS=200

5) コンピュータを再起動します. DoubleSpace はドライブを取り付けることができ
   るはずです.

それでもドライブが取り付けられないときは, 次の操作を行います.

1) ScanDisk を使ってお使いのハード ディスクの信頼性をチェックします. それに
   は, MS-DOS のコマンドプロンプトに次のように入力して[Enter]キーを押します.

   SCANDISK /ALL /SURFACE

2) コンピュータを再起動します. ドライブが取り付けられない場合は, 手順 3 以下
   を行います.

3) DBLSPACE.000 ファイルから, 隠しファイル, システムファイル, および書き込み
   禁しファイルとしての属性を解除します. たとえば, 圧縮ボリュームファイルが
   H:\DBLSPACE.000 である場合は, MS-DOS のコマンドプロンプトに次のように入力
   して[Enter]キーを押します.

   ATTRIB H:\DBLSPACE.000 -H -S -R

4) Microsoft Defragmenter を実行します. それには MS-DOS のコマンドプロンプト
   に DEFRAG と入力して[Enter]キーを押します.

5) ATTRIB コマンドを使って DBLSPACE.000 ファイルの属性を元に戻します. たとえ
   ば, 圧縮ボリューム ファイルが H:\DBLSPACE.000 である場合は, MS-DOS のコマ
   ンドプロンプトに次のように入力して[Enter]キーを押します.

   ATTRIB H:\DBLSPACE.000 +H +S +R

6) コンピュータを再起動します. DoubleSpace はドライブを取り付けることができ
   るはずです.

6.15 圧縮されたフロッピーディスクから書き込み禁止を取り除く
-----------------------------------------------------------
書き込み禁止の圧縮フロッピーディスクを使っていて, 自動取り付けがオンになって
いる場合, そのディスクから書き込み禁止の属性を解除しても, 取り外されるまでは
書き込み禁止のままになります.

書き込み禁止を取り除くには, 次のいずれかの操作を行います.

*  DBLSPACE /UNMOUNT コマンドを使ってフロッピーディスクを取り外してから, 再
   度そのディスクを使用します (この場合ディスクは自動的に再び取り付けられま
   す). たとえばディスクがドライブ B に挿入されていれば, DBLSPACE /UNMOUNT B:
   と入力して[Enter]キーを押してから, 再度そのディスクを使用します. (Windows
   が起動しているときは, DBLSPACE /UNMOUNT コマンドを使わないでください. そ
   の場合は, 次の手順に従います).

*  他のフロッピーディスクをドライブに挿入してアクセスします(たとえば DIR コ
   マンドを実行するなど). これにより, それまでそのドライブに挿入されていた圧
   縮フロッピーディスクは自動的に取り外されます. そのあと, 圧縮フロッピーディ
   スクを挿入し, 再度そのディスクを使用します(この場合ディスクは自動的に再び
   取り付けられます).

6.16 DoubleSpace でドライブを圧縮した後にエラーメッセージが表示される場合
-------------------------------------------------------------------------
DoubleSpace がシステム上のドライブを圧縮した後, 1 つまたは複数のドライブでディ
スクエラーが発生しましたというメッセージや, 新しく作成されたドライブの空き領
域は 0 Byteですというメッセージが表示されることがあります. たとえば,
DoubleSpace がドライブ C を圧縮した後, 次のようなエラーメッセージが表示され
ることがあります. (それぞれ[OK]ボタンを選択すると順に表示されます).

  DoubleSpace は, このドライブの圧縮を終了しました.
  しかし, ディスク領域が足りなかったためファイルは圧縮されていません.
  詳細は, [F1]キーを押してください.

  DoubleSpace はドライブ C: を調べることができませんでした.
  ドライブ C: にはディスクエラーが発生しています. 修復するには, コマンドプロ
  ンプトで SCANDISK C: と入力して[Enter]キーを押してください.

  DoubleSpace は, ドライブ H: を取り付けることができませんでした.
  ドライブ H: にはディスクエラーが発生しています. 修復するには, コマンドプロ
  ンプトで SCANDISK H: と入力して[Enter]キーを押してください.

  DoubleSpace はドライブ H: を調べることができませんでした.
  この操作を続けるには, コンベンショナル メモリが足りません.
  メモリを増やす方法は, 『MS-DOS 6.2/V  ユーザーズ ガイド』を参照してくださ
  い.

  DoubleSpace は, 新しくドライブ H: を作成しました.
  そのドライブには, 0.0MB の圧縮されていない領域があります.

DoubleSpace は, DBLSPACE.BIN がメモリに読み込まれたとき, 削除セントリの存在
だけをチェックしています. したがって, 最初の DBLSPACE ドライブを圧縮するとき,
削除セントリによって上記のエラーが発生します.

この問題が発生するのは, DoubleSpace がディスクの空き領域をチェックするとき,
隠し属性を持つ削除セントリディレクトリを考慮しないためです.

DoubleSpace がコンピュータを再起動するとき, 削除セントリは読み込まれないので,
ドライブを圧縮するのに空き領域が足りないことになるのです.

この問題を解決するには, 次の手順に従います.

1) AUTOEXEC.BAT を編集して, UNDELETE /SENTRY の行を REM ステートメントにしま
   す. たとえば, 次のようにします.

   REM UNDELETE /SENTRY

2) AUTOEXEC.BAT を保存してコンピュータを再起動します.

3) DoubleSpace の圧縮解除を実行します. それには, MS-DOS のコマンドプロンプト
   で DBLSPACE /UNCOMPRESS C: と入力して[Enter]キーを押します.

4) ドライブ C を圧縮し直します. それには, MS-DOS のコマンドプロンプトで
   DBLSPACE /COMPRESS C: と入力して[Enter]キーを押します.

5) AUTOEXEC.BAT ファイルを編集し, UNDELETE /SENTRY 行の REM コマンドを削除し
   ます.

6) AUTOEXEC.BAT ファイルを保存してコンピュータを再起動します.

6.17 DoubleSpace が無限にドライブを圧縮し続ける場合
---------------------------------------------------
ドライブ C を圧縮しているとき, DoubleSpace がドライブを最適化しようとして,
ドライブを圧縮し続ける無限ループに陥ってしまうことがあります.

この問題は, ファイルアロケーションテーブル(FAT)内で未使用クラスタと見なされ
たクラスタが, ドライブ内に数多くあるときに発生します. この問題が生じるときは,
未使用クラスタがディスク全体に分散して, ドライブを不連続化しているはずです.
各フラグメント内のファイルが最適化されていても, この問題は発生します.

DoubleSpace が圧縮ボリュームファイル(CVF)のサイズを拡大しようとするとき, ド
ライブに不連続な未使用クラスタがあまりにも多すぎると, サイズを拡大できません.
DoubleSpace は, ドライブが過度に不連続化されているものと見なして, Defrag を
実行してしまいます.

DoubleSpace は, DBLSPACE.INI ファイルの MaxFileFragments の値を増やすことで
この状況を修復できるのですが, この問題の場合は不連続化のために生じたものと見
なしてしまうので, 状況を修復できないのです. Defrag は高度にドライブを最適化
することができるので(多くの未使用クラスタがある場合), エラーを返しません(エ
ラーが返されれば DoubleSpace は MaxFileFragments の値を増やします). 分散して
いるクラスタは不良クラスタと見なされるため, DoubleSpace にとってはフラグメン
トが多すぎて CVF のサイズを拡大することができないわけです.

この問題を解決するには, 次の手順に従います.

1) DBLSPACE.INI のファイル属性を次のように変更します.

  ATTRIB -S -R -H DBLSPACE.INI

2) テキスト エディタなどを使って, DBLSPACE.INI を開きます.

3) MaxFileFragments の値を 2000以上に増やします.

4) DBLSPACE.INI を保存し, ファイル属性を元に戻します.

  ATTRIB +S +R +H DBLSPACE.INI

5)コンピュータを再起動します.

DoubleSpace がドライブを圧縮し終えたら, ScanDisk を実行してください. ディス
クが未使用クラスタで占有されている場合は, 毎日 ScanDisk を実行することをおす
すめします. それでもそのディスクが使用できない場合は, メーカーにお問い合わせ
ください.

6.18 ドライブを取り付けできないというメッセージが表示される場合
---------------------------------------------------------------
DoubleSpace を使って, 書き込み禁止ディスクを取り付けようとすると, 次のような
エラーメッセージが表示されます.

   <x>:\DBLSPACE.nnn は有効な圧縮ボリュームファイルではないので
   DoubleSpace はドライブ <x> を取り付けることができません.

<x> は, フロッピー ディスク ドライブ名です.

MS-DOS 6.2/V の環境下で, 書き込み禁止のリムーバブルメディアディスクを取り付
けようとすると, 次のようなメッセージが表示されます.

DoubleSpace はドライブ <x> を取り付けることができません.
<x>:\DBLSPACE.000 ファイルは有効な圧縮ボリュームファイルではありません.この
問題を処理するには, MS-DOS のコマンドプロンプトで SCANDISK X:\DBLSPACE.000
と入力してください.

<x> は, フロッピー ディスク ドライブ名です.

この問題を解決するには, ディスクの書き込み禁止を解除してください.

DoubleSpace は, 圧縮ボリュームファイル(CVF)を取り付けるとき,
Microsoft DoubleSpace BIOS パラメータブロック(MDBPB)(環境設定セクタの全部)と
BITFAT(入手可能なヒープセクタのテーブル)がカレントになるようにします. それに
は, DoubleSpace がディスクに書き込みを行う必要があります. この機能は現在
チェック中です.

DoubleSpace を実行してフロッピーディスクを取り付けることもできますし, MS-DOS
のコマンドプロンプトで DBLSPACE /MOUNT コマンドを使って, フロッピーディスク
を取り付けることもできます. たとえば, ドライブ A がフロッピーディスクドライ
ブの場合は, MS-DOS のコマンドプロンプトで次のように入力して[Enter]キーを押し
ます.

   DBLSPACE /MOUNT A:
  
6.19 DoubleSpace の圧縮率
-------------------------
ここでは, DoubleSpace の実質圧縮率(ACR)と推定圧縮率(ECR), および圧縮率の表示
と変更に関する問題について説明します.

実質圧縮率
----------
実質圧縮率(ACR)は, ドライブ上のデータの圧縮率を示します. ACR は, ハードディ
スクに保管するファイルの種類に応じて, 刻々と変化します.

ACR は, 圧縮されていないデータの合計を, データ圧縮後のディスクの使用領域の大
きさで割ることによって算出されます. たとえば, 圧縮されていないデータが 100MB
あるディスクを圧縮した結果ディスクの占有領域が 50MB になった場合,
圧縮率は 2:1 です. 公式は次のとおりです.

   100MB/50MB=2.0

注意
画面では, 2.0:1 と表示されます.

推定圧縮率
----------
推定圧縮率は, DoubleSpace が圧縮ドライブ上で空き領域の大きさを計算するのに使
用する数値です. ECR を変更しても, 圧縮アルゴリズムや ACR は変わりません. 単
に, DoubleSpace がディスクの空き領域を推測するのに使用する値を変わるだけです.

高度に圧縮可能なファイルを保管するときで, ディスク領域が少ない場合は, 圧縮率
を拡大します.

ECR の詳細については, DoubleSpace を実行して, [ドライブ]メニューから
[圧縮率変更]コマンドを選択して[F1]キーを押してください. または, MS-DOS のコ
マンドプロンプトで HELP DBLSPACE /RATIO と入力して[Enter]キーを押してくださ
い.

圧縮率の表示と変更
------------------
ドライブの ACR と ECR を確認するには, MS-DOS のコマンドプロンプトで DBLSPACE
と入力し, そのあとに目的のドライブ名とコロン(:)を入力して[Enter]キーを押しま
す.

ファイルやディレクトリの ACR を確認するには, DIR /C あるいは DIR /CH コマン
ドを使います. たとえば, COMMAND.COM の ACR を確認するには, MS-DOS のコマンド
プロンプトで次のように入力して[Enter]キーを押します.

   DIR C:\DOS\COMMAND.COM /CH

DOS ディレクトリの ACR を確認するには, MS-DOS のコマンドプロンプトで次のよう
に入力して[Enter]キーを押します.

   DIR C:\DOS /CH

注意
DIR /C コマンドは, 8KB のクラスタサイズに基づいて, 圧縮されたデータの ACR を
計算します. DIR /CH コマンドは, ホストドライブのクラスタサイズに基づいて, 圧
縮されたデータの ACR を計算します. ACR と ECR を比較するときは, この数値を使
用してください.

ECR を ACR の値に設定するには, MS-DOS のコマンドプロンプトで次のように入力し
て[Enter]キーを押します.

   DBLSPACE /RATIO=N.N

N.N は, ECR の値です.

注意
ホストドライブのサイズを変更することは, ECR を, 既定の値(2.0:1)から ACR の値
に変更することです.


6.20 DoubleSpace を解除してファイルを保存する
---------------------------------------------
MS-DOS 6.2/V をお使いの場合は, DoubleSpace を起動して, [ツール]メニューから
[圧縮解除]コマンドを選択してください. 圧縮ドライブを解除するときに問題が生じ
た場合は, 以下に説明する手順に従ってください.

この手順は, ファイルが保存されているハードディスクから DoubleSpace を削除す
る方法を示しています. 長い手順ですので, DoubleSpace を削除する必要がある場合
だけ実行してください. 圧縮ドライブのサイズを変更する必要がある場合は, MS-DOS
のコマンドプロンプトで, HELP DBLSPACE と入力して[Enter]キーを押し, オンライ
ンヘルプの DBLSPACE /SIZE というトピックを参照してください.

DoubleSpace(および圧縮ドライブに保存されているすべてのデータ)を削除する場合
で, ファイルを保存しておく必要がないとき, 簡単に圧縮ドライブを削除する方法が
あります. それには, DoubleSpace または MS-DOS のコマンドプロンプトで, 圧縮ボ
リュームファイル(CVF)を削除します. DoubleSpace で圧縮ボリュームファイル(CVF)
を削除するには, [ドライブ]メニューから[削除]コマンドを選択します.

注意
以下に挙げる例では, 起動ドライブ(ドライブ C)を圧縮していて, DoubleSpace のホ
ストドライブが H ドライブであることを想定しています.

起動ドライブから DoubleSpace を削除しない場合は, 手順 7) をとばしてください.

1) Microsoft Backup あるいは他社製のバックアップ ユーティリティを使って, 保
   存しておきたいすべてのファイルを, 圧縮ドライブ(ドライブ C)からバックアッ
   プしてください. (Microsoft Backup の使い方については, 『MS-DOS 6.2/V ユー
   ザーズガイド』の ｢4.1 ファイルのバックアップ:Backup」を参照してください).

2) DoubleSpace を削除すると, 現在のドライブ H はドライブ C になります. この
   ことは, 現在はドライブ H から起動していることを意味します. ドライブ H か
   ら起動でき, かつバックアップ ファイルを復元できるためには, DoubleSpace の
   ホストドライブには, 必要な MS-DOS のシステムとユーティリティが含まれてい
   なければなりません. さらに, ネットワークドライブにバックアップファイルを
   保管する場合は, ネットワークのリダイレクタが使用可能になっている必要があ
   ります.

MS-DOS のファイル(およびネットワークのリダイレクタ)をホストドライブ(ドライブ
H)にコピーするのに必要な空き領域を指定します. 必要な空き領域の大きさを決める
には, DIR コマンドを使います. たとえば, MS-DOS のファイルがどれくらいの領域
を必要とするかを確認するには, MS-DOS のコマンドプロンプトで DIR C:\DOS と入
力して[Enter]キーを押します.

次のような情報が画面に表示されます.

...(省略)
UNFORMAT.COM
VALIDATE.COM
VSAFE.COM
XCOPY.EXE
    194 個   7,003,143 バイトのファイルがあります
            12,959,744 バイトの空きがあります

最後から 2 番目の行は DOS ディレクトリのファイルによって使用されているバイト
数を表示しています. この数値が, DoubleSpace を削除したあと, 必要なファイルと
ユーティリティを保存するのに必要なディスク領域の大きさになります.

3) DoubleSpace の圧縮ボリュームから未使用ディスク領域を解放するには, MS-DOS
   のコマンドプロンプトで DBLSPACE /SIZE と入力して[Enter]キーを押します.

4) ホストドライブに空き領域がどれくらいあるか確認します. それには, ドライブ H
   に変更して, DIR コマンドを使います. DIR コマンドを実行すると表示される情
   報の最後の行は, ドライブ H の空き領域のバイト数を表しています. この数値が
   手順 2 で確認した数値よりも大きい場合は, ファイルやユーティリティをコピー
   するのに十分な空き領域がありますので, 手順 6 へ進んでください. ホストドラ
   イブに十分な空き領域がない場合は, 手順 5 へ進んでください.

5) 手順2 で確認した大きさの領域を確保するために, ドライブ C の不要なファイル
   を削除します. (注意:MS-DOS やネットワークのファイルは削除しないでください.
   この手順を実行するのに必要なファイルです.)

   不要なファイルを削除するには, DELTREE コマンドを使用することもできます
   (DELTREE はディレクトリ全体を削除します). たとえば, WORD ディレクトリとそ
   の中にあるすべてのファイルとサブディレクトリを削除するには, MS-DOSのコマ
   ンドプロンプトで DELTREE /Y C:\WORD と入力して[Enter]キーを押します.

ファイルを削除したら, MS-DOS のコマンドプロンプトで DBLSPACE /SIZE と入力し
て[Enter]キーを押し, 再び DoubleSpace ボリュームを圧縮します.

十分な空き領域を確保できたかどうか確認するには, ドライブ H に変更して, DIR
コマンドを使います. 使用領域のバイト数と空き領域のバイト数が表示されます. 十
分な空き領域が確保できたら, 手順 6 に進んでください. 空き領域が足りない場合
は, 手順 5 を繰り返してください.

6) MS-DOS とネットワークの必要なファイル(および手順 2 で必要と判断したファイ
   ル)をすべて, ホストドライブ(ドライブ H)にコピーします. ファイルとディレク
   トリ構造を保持したままコピーするには, XCOPY コマンドに /S スイッチを付け
   ます. たとえば, MS-DOS のファイルをすべてドライブ H の DOS ディレクトリに
   コピーするには, 次のように入力して[Enter]キーを押します.

   MD H:\DOS
   XCOPY C:\DOS\*.* H:\DOS

7) ホストドライブのルートディレクトリに COMMAND.COM がコピーされているか確認
   します. それには, MS-DOS のコマンドプロンプトに次のように入力して[Enter]
   キーを押します.

   DIR H:\COMMAND.COM

   COMMAND.COM がない場合は, 起動ドライブ(ドライブ C)から次のコマンドを使っ
   てコピーしてください.

   COPY C:\COMMAND.COM H:\

   AUTOEXEC.BAT と CONFIG.SYS ファイルに対してもこの手順を繰り返してください.
   これらのファイルはホストドライブのルートディレクトリに必要なものです.

   以上の操作で, 圧縮されていないドライブから起動したり, バックアップファイ
   ルを復元したりするのに必要なファイルがすべて揃いました. これで,
   DoubleSpace ボリュームを削除することができます.

8) ホストドライブのルートディレクトリに移動します. それには, MS-DOS のコマン
   ドプロンプトで次のように入力して[Enter]キーを押します.

   H:
   CD \

9) 次のコマンドを使用して, DoubleSpace 関連のファイルを削除します.

   DELTREE /Y DBLSPACE.*

10) 起動ドライブから DoubleSpace を削除する場合は, MS-DOS Editor などのテキ
    ストエディタを使って, ホストドライブ(ドライブ H)から CONFIG.SYS ファイル
    を開きます. 起動ドライブから DoubleSpace を削除しない場合は, ドライブ C
    から CONFIG.SYS ファイルを開きます. CONFIG.SYS ファイル内の DBLSPACE.SYS
    を参照している箇所を削除します. たとえば, DBLSPACE.SYS 読み込むための
    DEVICE コマンドを次のように変更します.

    REM DEVICE=C:\DOS\DBLSPACE.SYS

11) [Ctrl]+[Alt]+[Del]キーを押して, コンピュータを再起動します.

12) バックアップファイルを復元します.

これで, DoubleSpace はシステムから削除されました.

6.21 ブロックデバイスドライバとホストドライブ
---------------------------------------------
DoubleSpaceのホストドライブは，CONFIG.SYS 内のブロックデバイス数によって決定
されます.  
*　CONFIG.SYS ファイル内のブロックデバイスドライバがドライブ Z まで使用した場
   合には, DoubleSpace はホストドライブを割り当てることができず, 圧縮ドライブ
   が使用できなくなります. この現象が発生した場合には, コンピュータを再起動し
   "Starting MS-DOS..." と表示されている場合に, [F8]キーを押し, 追加したブロッ
   クデバイスをスキップしてください. MS-DOS のコマンドプロンプトが表示された
   ら, CONFIG.SYS ファイルを編集し, 追加したブロックデバイスドライバを削除し
   て, コンピュータを再起動します.
*  DoubleSpace の予約ドライブを, ドライブ Z まで使用した場合には,DEVICEHIGH=
   C:\DOS\DBLSPACE.SYS /MOVE 以降に設定したデバイスドライバは, エラーとなるの
   で注意してください.
*  Windows のスワップファイルをホストドライブに設定している場合には, ブロック
   デバイスドライバをDEVICEHIGH=C:\DOS\DBLSPACE.SYS /MOVEより前に追加すると
   Windows のスワップファイルのエラーが発生します.この場合には Windows のス
   ワップファイルを変更し，ブロックデバイスドライバを組み込むか,または 
   DEVICEHIGH=C:\DOS\DBLSPACE.SYS /MOVE 以降にブロックデバイスドライバを組み込
   むようにしてください.

6.22 DoubleSpace が使用するメモリを減らす
-----------------------------------------
高速セットアップ等で, DoubleSpace を組み込んだ場合には, DoubleSpace はホスト
ドライブ用に 5 台分(1 台を使用)の予約ドライブ領域を確保します. もし, メモリ
の空き領域が少しでも必要な場合には, 次の手順によりメモリ領域を少し拡大するこ
とができます.
予約ドライブ 1台につき, 約 120 byte のメモリが必要となります. また,
DoubleSpace の既定の設定では, 自動取り付け機能がオンの状態となっています
(約 4 KB を使用). この機能もオフにすれば, メモリをさらに増加できます.

予約ドライブ領域の開放方法

1) MS-DOS のコマンドプロンプトで, DBLSPACE /LIST と入力し, 圧縮ドライブのホ
   ストドライブを確認します.

   C:\>DBLSPACE /LIST
   ﾄﾞﾗｲﾌﾞ 種類			      全空き	  全ｻｲｽﾞ     CVF ﾌｧｲﾙ名
   -----  --------------------------   ----------  ---------  ----------
     A    着脱可能メディアドライブ     ドライブにディスクがありません
     C    圧縮ハードドライブ		19.8 4MB    32.06MB  H:\DBLSPACE.000
     D    DoubleSpace に使用可能
     E    DoubleSpace に使用可能
     F    DoubleSpace に使用可能
     G    DoubleSpace に使用可能
     H    ローカルハードドライブ	 0.82 MB    23.22 MB

   この状態は, 圧縮ドライブ C: のホストドライブが H: ドライブに割り当てられ
   ている事を示します.

2) MS-DOS のコマンドプロンプトで, DBLSPACE /HOST=D: H: と入力し, ホストドラ
   イブ H: を D: ドライブに変更し, コンピュータを再起動します.

3) 起動後, MS-DOS のコマンドプロンプトで, DBLSPACE /LIST と入力し, 圧縮ドラ
   イブのホストドライブを確認します.

   C:\>DBLSPACE /LIST
   ﾄﾞﾗｲﾌﾞ 種類			      全空き	  全ｻｲｽﾞ     CVF ﾌｧｲﾙ名
   -----  --------------------------   ----------  ---------  ----------
     A    着脱可能メディアドライブ     ドライブにディスクがありません
     C    圧縮ハードドライブ		19.8 4MB    32.06MB  H:\DBLSPACE.000
     D    ローカルハードドライブ	 0.82 MB    23.22 MB
     E    DoubleSpace に使用可能
     F    DoubleSpace に使用可能
     G    DoubleSpace に使用可能
     H    DoubleSpace に使用可能

   この状態は, 圧縮ドライブ C: のホストドライブが D: ドライブに割り当てられ
   ている事を示します.

4) MS-DOS のコマンドプロンプトで, DBLSPACE /LASTDRIVE=D: と入力し, DBLSPACE
   の予約ドライブ H: を D: に変更し, コンピュータを再起動します.

5) EMM386 が組み込まれている場合には, MemMaker を MS-DOS のコマンドプロンプ
   トで実行します.


自動取り付け機能の OFF

1) MS-DOS のコマンドプロンプトで, DBLSPACE /LIST と入力し, 自動取り付け機能
   を確認します.
   "全ドライブの自動取り付けは無効です." と表示されている場合にはこれ以上変
   更できません.

2) MS-DOS のコマンドプロンプトで, DBLSPACE /AUTOMOUNT=0 と入力し, 自動取り付
   け機能をオフにし, コンピュータを再起動します.

3) EMM386 が組み込まれている場合には, MemMmaker を MS-DOS のコマンドプロンプ
   トで実行します.

6.23 1024BPS フロッピーディスクの圧縮
-------------------------------------
DoubleSpace では, 1024BPS のフロッピーディスクは圧縮できません. 1024BPS のフ
ロッピーディスクをフロッピーディスクドライブに挿入している状態で DoubleSpace
を起動し[圧縮]メニューの[既存ドライブ]コマンドを選択しても, 1024BPS のフロッ
ピーディスクは，表示されません. また，他社製の 3 モードドライバによって認識
されている場合にも圧縮しないでください. 最悪の場合 ハードディスクが壊れる場
合があります.

6.24 ホストドライブが圧縮時と異なる
-----------------------------------
DoubleSpace では取り付けられていないホストドライブを, DoubleSpace のメイン画面
から取り付けを行うと, 異なるドライブに割り当てられることがあります. 
ホストドライブを指定したい場合には, MS-DOS のコマンドプロンプトで取り付けを行
います. 詳しくは MS-DOS Help の DBLSPACE /MOUNT を参照してください.

6.25 圧縮ドライブにディレクトリを作成すると空き領域がなくなってしまう
---------------------------------------------------------------------
DoubleSpace の圧縮ドライブにいくらかの空き領域がある(たとえば, 推定圧縮率 2:1
のときに空き領域が 128KB ある)のに, ディレクトリを作成すると, ディスクの空き
領域が 0 Byteになってしまうことがあります.

DoubleSpace は, 圧縮ドライブがほぼ一杯のときにもいくらかの空き領域を確保して
います. 特に, 圧縮ボリュームファイルの空き領域が, 非圧縮領域の境界である 64KB
未満になった場合, DoubleSpace は残りの空き領域をすべて自身のために確保します. 

たとえば, 推定圧縮率(ECR)が 2.0:1 で, ドライブの空き領域が 130KB と表示され
たとき, 実際には非圧縮セクタには 65KB しか残っていません. ディレクトリを作成
すると, 16 セクタ, つまり 8KB 消費してしまうので, 非圧縮領域は 64KB 未満になっ
てしまいます. その結果, DoubleSpace は残りの空き領域を自身のために確保してし
まい, ディスクの空き領域は 0 Byteしかないと表示してきます. 

6.26 フロッピーディスクドライブのインジケータが点滅する
-------------------------------------------------------
MS-DOS 6.2/V 上で, 特定のソフトウェアを起動すると, フロッピーディスクドライ
ブのインジケータが点滅することがあります. 

この現象は, ソフトウェアが Int 21h Function 44h Get Device Parameter IOCTL
を呼び出すときに発生します. この IOCTL のために, DoubleSpace の自動取り付け
コードが, フロッピーディスクドライブが圧縮フロッピーディスクであるかどうかを
チェックしてしまうのです. 

次に挙げるアプリケーションが, Int 21h Function 44h を呼び出すために上記の現
象を発生させます. 

   Norton Desktop for Windows
   Central Point PCTools Data Monitor(DATAMON.EXE)

この問題を解決するには, DoubleSpace の自動取り付けスイッチをオフにします. そ
れには, MS-DOS のコマンドプロンプトで DBLSPACE /AUTOMOUNT=0 と入力して[Enter]
キーを押し, 次に, [Ctrl]+[Alt]+[Del]キーを押してコンピュータを再起動します. 

6.27 Norton Disk Cashe ユーティリティの自動取り付け
---------------------------------------------------
Norton Cashe (NCACHE2.EXE) や Norton Speedrive(SPEEDRV.EXE) などの Norton
Disk Cashe ユーティリティをお使いの場合は, これらのユーティリティを読み込む
コマンドが, DBLSPACE.SYS を読み込むコマンドより後に記述されているようにして
ください. これらのユーティリティのいずれかを読み込むコマンドが, DBLSPACE.SYS
を読み込むコマンドより前に記述されている場合は, DoubleSpace の自動取り付け機
能は正常に動作しません.
